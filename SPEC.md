# フリマ会計CSV出力 Chrome拡張機能 仕様書

## 概要

フリマサイトの販売履歴を読み取り、会計SaaS（freee / マネーフォワードクラウド）向けの仕訳インポートCSVを出力するChrome拡張機能。

- **現フェーズ対応サイト**: メルカリ
- **将来対応予定**: ラクマ、PayPayフリマ
- **出力形式**: freee仕訳インポートCSV / マネーフォワードクラウド仕訳CSVを選択式

---

## 対象ユーザー

- 個人事業主
- 課税事業者（インボイス登録の有無は問わない）
- フリマサイトの売上を事業所得として申告している方

---

## 機能一覧

### 1. データ取得（メルカリ）

#### 対象ページ
```
https://jp.mercari.com/mypage/listings/sold?year={年}
```

#### 取得データ項目
| HTMLの列     | 用途                     | 備考                    |
|------------|------------------------|-----------------------|
| 購入完了日      | 仕訳日付（取引日）              |                       |
| 商品タイトル     | 摘要                     |                       |
| 商品価格       | 売上高の金額（税込）             |                       |
| 販売手数料（税込）  | 支払手数料の金額（税込）           | 原則10%（メルカリ手数料）        |
| 送料（税込）     | 荷造運賃の金額（税込）            |                       |
| 他費用（税込）    | 広告宣伝費の金額（税込）           | `---` の場合はスキップ        |
| 税率         | 売上高の消費税率               | 「10%」→「課税売上 10%」に変換   |
| 販売利益       | 売掛金（借方）の金額             | 商品価格 - 手数料 - 送料 - 他費用 |

- 「寄付」列は取得・処理しない。

#### ページネーション
- 1ページ20件。次ページURL: `?year={年}&page={n}`
- 拡張機能が自動でページを順次取得し、全件ロードする。
- ユーザーはロード完了後に一覧を確認・絞り込みできる。

---

### 2. 月フィルター

- ページネーション完了後、全取引一覧をポップアップに表示。
- ユーザーが「月」を選択すると、該当月の取引に自動でチェックが入る。
- チェックは手動でON/OFFも可能（個別除外・追加のため）。
- チェックが入っている取引のみCSV出力対象となる。

---

### 3. 仕訳ルール

#### 仕訳パターン（1取引 = 複数行の複合仕訳）

1件の売上を以下の複合仕訳で表現する。

| 行  | 借方勘定科目   | 借方金額    | 借方税区分    | 貸方勘定科目 | 貸方金額    | 貸方税区分       |
|----|----------|---------|----------|--------|---------|-------------|
| 1  | 売掛金      | 販売利益    | 対象外      | 売上高    | 商品価格    | 課税売上 10%    |
| 2  | 支払手数料    | 販売手数料   | 課税仕入 10% | （空）    | （空）     | （空）         |
| 3  | 荷造運賃     | 送料      | 課税仕入 10% | （空）    | （空）     | （空）         |
| 4  | 広告宣伝費    | 他費用     | 課税仕入 10% | （空）    | （空）     | （空）         |

- 行4（他費用）は `---` の場合は出力しない。
- 貸方空白行はfreee/MFクラウド両方でサポートされている複数行仕訳の記法に従う。

**借方合計 = 貸方合計（バランス検証）**
```
借方: 販売利益 + 手数料 + 送料 + 他費用 = 商品価格
貸方: 商品価格
```

#### 税額の計算
```
税額 = floor(税込金額 ÷ 11)   // 10%税率の場合
```
- 売上高の税額: `floor(商品価格 ÷ 11)`
- 仕入側（手数料・送料・他費用）の税額: `floor(各金額 ÷ 11)`

#### 摘要フォーマット
```
{商品タイトル}（メルカリ）
```

---

### 4. 勘定科目設定

設定画面でユーザーが変更可能。設定は `chrome.storage.sync` で永続保存。

| 設定項目   | デフォルト値  |
|--------|---------|
| 売上勘定科目（貸方） | 売上高     |
| 未収金勘定科目（借方） | 売掛金     |
| 手数料勘定科目（借方） | 支払手数料   |
| 送料勘定科目（借方）  | 荷造運賃    |
| 他費用勘定科目（借方） | 広告宣伝費   |

---

### 5. 出力形式の選択

- **freee** / **マネーフォワードクラウド** をUIで選択。
- 最後に選択したSaaSを `chrome.storage.sync` に保存し、次回起動時に復元する。

## CSV出力フォーマット

### freee 仕訳インポートCSV

文字コード: UTF-8（BOMなし）
参考: `ref/freee_仕訳インポート（UTF-8）.csv`

#### ヘッダー行（表題行）
```
[表題行],日付,伝票番号,決算整理仕訳,借方勘定科目,借方科目コード,借方補助科目,借方取引先,借方取引先コード,借方部門,借方品目,借方メモタグ,借方セグメント1,借方セグメント2,借方セグメント3,借方金額,借方税区分,借方税額,貸方勘定科目,貸方科目コード,貸方補助科目,貸方取引先,貸方取引先コード,貸方部門,貸方品目,貸方メモタグ,貸方セグメント1,貸方セグメント2,貸方セグメント3,貸方金額,貸方税区分,貸方税額,摘要
```

#### 明細行の例（商品価格1350円、手数料135円、送料210円）
```
[明細行],2026/02/28,1,,売掛金,,,,,,,,,,,1005,対象外,0,売上高,,,,,,,,,,,1350,課税売上10%,122,【2個セット】パーフェクトワン...（メルカリ）
[明細行],2026/02/28,1,,支払手数料,,,,,,,,,,,135,課対仕入10%,12,,,,,,,,,,,,0,,,0,
[明細行],2026/02/28,1,,荷造運賃,,,,,,,,,,,210,課対仕入10%,19,,,,,,,,,,,,0,,,0,
```

- 伝票番号: エクスポート対象内での連番（1始まり）
- 複数行が同一伝票番号 → freeeが複合仕訳として扱う

## 拡張機能の構成（Chrome Extension）

```
freemarket-csv-extension/
├── manifest.json           // Manifest V3
├── popup/
│   ├── popup.html          // メインUI
│   ├── popup.js
│   └── popup.css
├── options/
│   ├── options.html        // 勘定科目・SaaS設定画面
│   ├── options.js
│   └── options.css
├── content/
│   └── mercari.js          // メルカリページのデータ抽出
├── background/
│   └── service_worker.js   // ページネーション自動読み込み制御
└── lib/
    ├── csv_freee.js        // freee CSV生成
    ├── csv_mf.js           // MFクラウド CSV生成
    └── journal.js          // 仕訳ロジック共通
```

### 各コンポーネントの責務

| ファイル | 責務 |
|---|---|
| `content/mercari.js` | DOMからテーブルデータを抽出しメッセージで返す |
| `background/service_worker.js` | 年・ページ指定でメルカリページへアクセスし全件収集 |
| `lib/journal.js` | メルカリデータ → 仕訳データへ変換（勘定科目マッピング・税額計算） |
| `lib/csv_freee.js` | 仕訳データ → freee CSV文字列生成 |
| `lib/csv_mf.js` | 仕訳データ → MFクラウド CSV文字列生成 |
| `popup/popup.js` | UI制御・年選択・月フィルター・チェック管理・CSV出力 |
| `options/options.js` | 勘定科目・SaaS選択の設定保存 |

---

## データフロー

```
[ユーザー操作]
  ↓ 年を選択 → 「読み込む」ボタン
[service_worker]
  ↓ page=1,2,3... を順次フェッチ
  ↓ 各ページの content script を実行してテーブルデータを抽出
  ↓ 全件を配列にまとめてポップアップへ返す
[popup.js]
  ↓ 全取引一覧を表示
  ↓ 月フィルター選択 → 該当月のチェックをON
  ↓ ユーザーがチェックを確認・調整
  ↓ 「CSVエクスポート」ボタン
[journal.js]
  ↓ チェック済み取引 → 仕訳データへ変換
[csv_freee.js or csv_mf.js]
  ↓ CSV文字列生成
[ブラウザ]
  → ファイルダウンロード
```

## 将来の拡張対応方針

各フリマサイトのスクレイパーを `content/` 配下に追加し、共通の仕訳ロジック（`lib/journal.js`）に渡す設計とする。

```
content/
├── mercari.js      // 現フェーズ実装
├── rakuma.js       // 将来
└── paypay_flea.js  // 将来
```

サイト固有の差異（URL・HTMLの構造・手数料率・送料の扱いなど）はスクレイパー側で吸収し、`journal.js` に渡すデータ構造は共通インターフェースとする。

### 共通取引データ型（内部表現）
```typescript
interface SaleRecord {
  platform: "mercari" | "rakuma" | "paypay";
  date: string;           // YYYY/MM/DD
  title: string;
  price: number;          // 商品価格（税込）
  commission: number;     // 販売手数料（税込）
  shipping: number;       // 送料（税込）
  otherCost: number;      // 他費用（税込）、なければ0
  profit: number;         // 販売利益
  taxRate: number;        // 0.10
}
```

## 未決事項・制約事項

- メルカリのHTMLはCSSクラス名がハッシュ化されているため、セレクタは `data-testid` 属性やテーブル構造（th/td の位置）を使用する。
- メルカリのHTML仕様変更によってスクレイパーが動作しなくなる可能性がある（保守が必要）。
- `chrome.storage.sync` の容量制限（100KB）の範囲内で動作する設計とする（取引データ自体は保存しない）。
- CSVのファイル名: `mercari_freee_YYYYMM.csv` / `mercari_mf_YYYYMM.csv`（対象月が複数の場合は `YYYY` のみ）。
